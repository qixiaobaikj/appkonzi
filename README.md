# 远程助手 — 帮助家中老人使用手机

一套专为"老人不会用手机"场景设计的安卓远程控制系统，家人安装一次后即可随时远程查看和操作老人的手机。

## 项目结构

```
appkonzi/
├── server/          # 信令服务器（Node.js）
├── agent/           # 被控端 App（安装到老人手机）
└── controller/      # 控制端 App（安装到家人手机）
```

---

## 快速开始

### 第一步：启动信令服务器（在家用电脑上运行）

> 要求：电脑和两部手机在**同一个WiFi**下

```bash
cd server
npm install
npm start
```

启动后会显示：
```
=======================================
  远程控制 信令服务器 已启动
  局域网地址: ws://192.168.1.xxx:8080
  本机地址:   ws://127.0.0.1:8080
  将以上地址填入App中即可连接
=======================================
```

记住 `局域网地址`（例如 `ws://192.168.1.100:8080`），后续填入App。

---

### 第二步：安装被控端 App（老人手机，一次性操作）

**打包方式 A：使用 GitHub Actions 云端打包（推荐，无需本地安装 Android Studio）**
1. 将项目推送到您自己的 GitHub 仓库。
2. GitHub 将自动触发构建。点击仓库顶部的 **Actions** 标签页。
3. 点击最新的运行工作流（如 `Build Android APKs`），滚动到最下方 **Artifacts** 区域，直接下载打包好的 `远程助手_被控端_Agent` 和 `远程控制_控制端_Controller`。
4. 解压下载的压缩包，将其中的 `.apk` 文件分别安装到老人手机和您的控制端手机。

**打包方式 B：使用 Android Studio 本地编译**
1. 用 Android Studio 打开 `agent/` 目录，**编译并安装**到老人手机。

5. 打开 App，进入授权引导界面：
   - **步骤1**：点击"开启无障碍服务" → 在系统设置中找到"远程助手"并开启
   - **步骤2**：点击"忽略电池优化" → 允许
6. 输入信令服务器地址（如 `ws://192.168.1.100:8080`）
7. 点击"启动服务"
8. ✅ 完成！老人无需再操作任何东西

> 手机重启后 App 会**自动启动**，无需再次操作。

---

### 第三步：安装控制端 App（家人手机）

**打包方式 A**：同上，从 GitHub Actions 下载解压直接安装。

**打包方式 B**：用 Android Studio 打开 `controller/` 目录编译安装。

1. 打开 App，点击右上角"+" → 输入信令服务器地址
2. 在设备列表中看到老人手机后，点击"连接"
3. 即可实时看到老人手机屏幕，并远程触控操控

## 功能说明

### 被控端（老人手机）
| 功能 | 说明 |
|------|------|
| 一次授权 | 只需开启无障碍服务，无其他复杂操作 |
| 开机自启 | 手机重启后自动恢复服务 |
| 防后台杀死 | 常驻通知保活，不被系统清理 |
| 实时投屏 | 10fps JPEG压缩，低延迟 |
| 接受操控 | 支持点击、滑动、长按 |

### 控制端（家人手机）
| 功能 | 说明 |
|------|------|
| 多设备管理 | 可添加多个被控设备 |
| 实时画面 | 显示老人手机屏幕 |
| 触控操控 | 在画面上点击/滑动即可操控老人手机 |
| 系统按键 | HOME键、返回键、最近任务键 |
| 延迟显示 | 实时显示网络延迟 |

## 技术架构

```
控制端App ←──WebSocket──→ 信令服务器 ←──WebSocket──→ 被控端App
                                                         │
                                           AccessibilityService
                                           (截图 + 触控注入)
```

### 屏幕捕获方案
使用 Android 11+ 的 `AccessibilityService.takeScreenshot()` API：
- ✅ **无需每次重启重新弹授权框**（与MediaProjection不同）
- ✅ 无障碍服务开机后自动恢复
- 截图 → 缩放到720p → JPEG压缩(40%) → Base64 → WebSocket发送

## 部署到云服务器（局域网调试好后）

```bash
# 在云服务器上
git clone <your-repo>
cd server
npm install
# 推荐用pm2保持运行
npm install -g pm2
pm2 start index.js --name "remote-control"

# App中将 ws://192.168.1.x:8080 改为 ws://你的服务器IP:8080
```

> 如果云服务器有防火墙，记得开放 8080 端口（TCP）

## 常见问题

**Q: 老人手机重启后还能用吗？**  
A: 可以。被控端有开机自启，服务会自动恢复。无障碍服务权限也会保留，无需重新授权。

**Q: 在4G/5G网络下能用吗？**  
A: 局域网版本仅支持同一WiFi。要在外网使用，需将服务器部署到云服务器。

**Q: 支持哪些Android版本？**  
A: 被控端需要 Android 11（API 30）或以上。控制端支持 Android 8.0+。

**Q: 会影响老人手机性能吗？**  
A: 截图和压缩在后台进行，CPU占用较低。仅在有控制端连接时才进行持续截图。
